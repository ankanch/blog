加速Android Studio的Gradle构建速度

Veröffentlicht am 2015-09-06 | in Android开发 / Android Studio

在利用Android Studio做项目时，发现随着项目内资源的逐渐增多（或者项目创建时间太过久远，而又未经常打开），Android Studio的build速度也越来越慢。（P.S.在做我的CSGO StatTrak的时候，基本要12分钟，才能build完成，在此期间，一直显示Gradle运行中….）

在互联网上一番搜索后，发现通过以下方法可以加快Gradle的构建速度。

在以下Gradle目录创建gradle.properties文件

  • C:Users&lt;username&gt;.gradle

    并在文件中增加以下数据，然后保存。
    org.gradle.daemon=true
    org.gradle.jvmargs=-Xmx2048m -XX:MaxPermSize=512m -XX:+HeapDumpOnOutOfMemoryError -Dfile.encoding=UTF-8
    org.gradle.parallel=true
    org.gradle.configureondemand=true

注意，因为这样配置是对于此计算机的Android Studio用户，所以对于所有项目都有效。

不过最好把Android Studio的配置也改改，打开Android Studio，选择菜单项 File->Settings。

切换到下面的视图：

把 Offline Work打上勾。

 

 

 

 

再切换到如下视图：

把下图黄色箭头指出的选项弄成一样

最后，保存，并重新打开Android Studio.

不出意外，前面操作都正确的话，你现在Gradle应该有了很大的提升。

我在怎样修改后，每次Gradle构建，只要40秒左右，比以前的12分钟，快多了。

Android Studio修改应用程序版本号并将新版应用上传到市场可能会遇到的问题

Veröffentlicht am 2015-09-03 | in Android开发 / Android Studio

我在将我一个更新版本的应用发布至Google Play时，遇到了如下问题：

因为我之前的应用版本号设置为了1，所以这个应用的版本号不能为1，于是我在manifest.xml中改掉了版本号（如图），再次上传，却仍然是这个问题。

众所周知：修改应用程序的版本号要在manifest.xml文件中的节中添加以下2个属性：

android:versionCode=
android:versionName=

而我修改后却仍然提示版本为1，在结果一番搜索+查看文件后，发现，在对应Moudle的Gradle文件里面也有版本号，而这里的版本号却没有改变，仍然是1.于是，我讲Gradle文件中的版本号改成和manifest文件中的一样后，问题，果真解决了。



### versionCode和versionName说明：

（来自官方文档，译者: http://weibo.com/huangts）

android:versionCode — 整型, 相对于其他种类的版本号而言, 表示应用程序代码的版本. 其值是个整型,因此其他应用程序可以在程序编码中判断它, 例如检查升级或降级关系. 可以对其设置任何值, 然而应当确保每一个后续发布的应用程序使用更大的值. 尽管系统并不强制执行这种规则, 但是每个后续版本使用增加的数字值是标准作法. 典型做法是, 你发布的第一个版本的versionCode为1, 然后以后的每个发布版本单调递增, 而不管这个发布版本是否是一个主要或者次要的发布版本. 这表示 android:versionCode 值没有必要与用户可见的应用程序发布版本号有特别的相似之处. 同时,应用程序和发布平台不应将此版本值呈现给用户.

android:versionName — 字符串值,向用户显示的应用程序代码的发布版本号. 此值是一个字符串, 因此可以用.. 的形式描述应用程序版本, 或者以任何其他类型的绝对或相对版本号标识的形式. 相对于 android:versionCode, 除了让应用程序向用户显示它以外, 系统不打算将这个值用于任何内部使用目的. 应用发布平台可能会将 android:versionName 值解析出来显示给用户. 在manifest文件的元素中同时定义这些个版本相关属性.

通过修改adb端口，解决Android Studio无法识别真机

Veröffentlicht am 2015-08-25 | in Android开发 / Android Studio

注意：该方法只适用于adb端口呗占用导致的无法识别真机！

最近新换了windows10，在调试Android Studio程序时发现：Android Studio无法识别真机。

经过一番搜索，了解到是因为adb端口被占用，导致问题发生。

虽然adb的端口被占用了，但是占用端口的进程是必须启动的，不能被杀死，网上很多办法都说的是杀死占用端口的进程。尝试后发现这个方法并不适用于我。

所以在此推荐一个新的方法：通过修改adb的默认端口，来防止端口被占用。

方法如下：

新建一个环境变量，名字为_ANDROID_ADB_SERVER_PORT_，然后它的值设为一个新的端口号码，比如说9999。

再次启动Android Studio调试程序，adb就可以识别真机了。

如下图，现在可以识别真机了。

注意：输入adb nodaemon server可以查看哪个端口被占用了，没有端口呗占用，输入后会无反应。

Android Studio的一个从Web API获取字符串的Java类（转）

Veröffentlicht am 2015-07-23 | in Android开发 / Android Studio . 转载

源代码如下：

package com.akakanch.toolkit;

/**

  • Created by Administrator on 2015/7/7.
  • 用来操作从Web API获取数据
    */
    import java.net.URL;
    import java.net.URLConnection;
    import java.io.BufferedReader;
    import java.io.IOException;
    import java.io.InputStreamReader;

public class WebClient{

private static String _newLine = System.getProperty(“line.separator”);
public WebClient()
{ }
public String getContent(String url, String oriEncoding, String targetEncoding) throws IOException
{
URL u = new URL(url);
URLConnection uc = u.openConnection();
BufferedReader in;

if(oriEncoding == null || oriEncoding.length() == 0)
{
in = new BufferedReader(new InputStreamReader(uc.getInputStream()));
}
else
{
in = new BufferedReader(new InputStreamReader(uc.getInputStream(), oriEncoding));
}
String line;
StringBuilder sb = new StringBuilder();
while((line = in.readLine()) != null)
{
sb.append(line); sb.append(_newLine);
}
if(targetEncoding == null || targetEncoding.length() == 0)
{
return sb.toString();
}
return new String(sb.toString().getBytes(), targetEncoding);
}
}

使用方法：

WebClient wc = new WebClient();
String str = wc.getContent(“http://api.akakanch.com/?app=1", “utf-8”, null);

 

str里即为Web API返回的字符串

Android程序储存数据到系统目录

Veröffentlicht am 2015-07-21 | in Android开发 / Android Studio

有些时候我们需要将我们的android app的一些数据储存起来，以便下次app快速加载。Android系统为我们提供了2中储存方式：手机自带内存和SD卡。这里介绍如何将数据储存在手机自带内存。

先说明：储存在手机自带内存，路径是系统内存的
data/<your app package name>/<file name>下，并且当你的app被卸载后，这里的数据也会被删掉。储存在内部储存的数据只有你的app可以访问，用户也无法通过资源管理器访问（除非手机已经root）

通过查阅Android SDK文档可知：

需要用到openFileOutput()函数来向文件写入，用openFileInput()函数来从文件读取数据。下面来详细介绍：

 

openFileOutput()与openFileInput()

原型：
—————————————————————————
**public abstract FileOutputStream openFileOutput (String name, int mode)

**第一个参数 指定文件名（没有路径，只有文件名）
第二个参数 指定输出模式，可以是以下的任意一个：
MODE_APPEND 在原文件末尾追加数据
MODE_PRIVATE 默认模式，覆盖原有数据，并且只有本程序可以访问
MODE_WORLD_READABLE 全局可读取（危险，不建议）
MODE_WORLD_WRITEABLE 全局可写入（危险，不建议）

该函数返回一个FileOutputStream对象，注意该函数只可在Activity类和继承自Activity相关类中调用，至于如何在非Activity类，如Fragment类中调用，文章后面会给出说明

**public abstract FileInputStream openFileInput (String name)

参数指定了文件名，同上，没有路径
该函数返回一个FileInputStream对象，注意该函数只可在Activity类和继承自Activity相关类中调用，至于如何在非Activity类，如Fragment类中调用，文章后面会给出说明
————————————————————————— 特别说明：**这两个函数都需要处理FileNotFoundException，将其放入try…catch中即可，在catch中捕获FileNotFoundException异常。后文代码会给出例子。

向文件写入与读取数据：

利用openFileInput()或者openFileOutput()函数返回的FileInputStream或者FileOutputStream对象调用write()或read()函数进行写入与读取。最后再调用close()关闭文件流。

由于write()接受的参数是byte类型，所以我们要调用String.getBytes()函数来传递参数。

向文件写入数据的代码如下：

EditText et = (EditText) view.findViewById(R.id.editText_layout_file_ops_edit);
String str = et.getText().toString();
try {
    FileOutputStream fout = context.getApplicationContext().openFileOutput("buffer", context.MODE_PRIVATE);
    fout.write(str.getBytes());
    fout.close();
}
catch (FileNotFoundException e)
{ e.printStackTrace(); }
catch(IOException e)
{ e.printStackTrace(); }
  从文件读取数据的代码如下： 
String str = new String();
try {
    FileInputStream fin = context.getApplicationContext().openFileInput("buffer");
    int length = fin.available();
    byte[] buffer = new byte[length];
    fin.read(buffer);
    str = EncodingUtils.getString(buffer, "UTF-8");
    fin.close();
    EditText et = (EditText) view.findViewById(R.id.editText_layout_file_ops_edit);
    et.setText(str);
} catch (FileNotFoundException e) {
    e.printStackTrace();
} catch (IOException e) {
    e.printStackTrace();
}

Android Studio在切换Fragment显示不同的内容与NavigationDrawerFragment解析

Veröffentlicht am 2015-07-18 | in Android开发 / Android Studio

我在做一个app时，使用了Navigation Drawer Fragment，但却在更换显示内容（或者说更换显示的dialog）的时候遇到了问题。

后来才知道，是不能通过setContentView来设置layout的。必须通过切换（替换）fragment来实现显示内容的改变。

首先介绍下什么是Fragment：

Android运行在各种各样的设备中，有小屏幕的手机，超大屏的平板甚至电视。针对屏幕尺寸的差距，很多情况下，都是先针对手机开发一套App，然后拷贝,修改布局以适应其它设备。Fragment的出现就是为了解决这种问题。你可以把Fragment当成Activity的一个界面的一个组成部分，甚至Activity的界面可以完全有不同的Fragment组成，Fragment拥有自己的生命周期和接收、处理用户的事件，这样就不必在Activity写一堆控件的事件处理的代码了。更为重要的是，你可以动态的添加、替换和移除某个Fragment。

特别说明：

Fragment必须放在某个Activity中，但是它在app的所有Activity中是通用的。也就是说你可以把一个Activity中的Fragment加载到其它的Activity中。
